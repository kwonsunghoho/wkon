# MONC 디자인 원칙 & 팔레트

> 이 문서는 저장소에 포함되어 **어느 컴퓨터에서든** 적용된다. UI/디자인 작업 전 반드시 참고.
> MONC는 항공 승무원 준비생 대상 2주 챌린지 랜딩. **회원 99%가 모바일 유입 → 모바일 퍼스트(375px)가 최우선.**

## 팔레트 (2026-07-04 리브랜드 → 2026-07-14 웜 통일 리디자인)

| 역할 | 색 | 토큰 |
|---|---|---|
| 넓은 면·배경 | 베이지 `#E9E4D8` | `--bg` |
| 메인/서브 타이틀 · 전환 CTA | 오렌지 `#F27945` | `--action` |
| 장식 면·큰 디스플레이 액센트 | 코랄 `#F27945` | `--accent` |
| 큰 볼드 강조(24px+/굵은 18.66px+)·테두리·포커스링 | 딥 코랄 `#D9531F` | `--accent-dark` |
| **소형 텍스트(12~15px) 강조·링크·활성 칩 배경** | 딥 번트 오렌지 `#A33D14` | `--accent-ink` |
| 본문 텍스트 | 다크 뉴트럴 `#26221C` | `--text` |
| CTA 버튼 글씨 | 진한 잉크 `#2A1206` | `--action-ink` |

- **네이비(`#194192`)는 2026-07-14부로 폐기** — `--accent`/`--rose`가 코랄로 전환됨(tokens.css 하단 오버라이드 블록). 커뮤니티 섹션의 `--mc-*` 서브테마(아이보리-네이비-골드)만 의도적 예외.
- ⚠️ **오렌지·코랄(`#F27945`)은 흰 글씨 대비가 낮다(≈2.7:1).** CTA/코랄 채움 = **진한 글씨(`--action-ink`, ≈5.6:1)**. 흰 글씨가 필요한 채움(활성 탭·버튼)은 `--accent-ink` 배경(흰 글씨 6.5:1).
- ⚠️ **소형 텍스트에 코랄·`--accent-dark` 금지** — `#D9531F`도 베이지 위 ≈3.2:1로 4.5:1 미달. 12~15px 강조는 반드시 `--accent-ink`(#A33D14 — 베이지·bg2·bg3·틴트·아이보리 전부 4.5:1+ 검증값).
- 섹션 타이틀 = 오렌지 + **명조(Noto Serif KR 700, `--fs-h2` 30~46px)**, 에이브로우 라벨(`.section-label`) = `--accent-ink` 솔리드. 명조 쓰는 페이지는 `<head>`에 Noto Serif KR `600;700;900` 링크 필수.
- 배경/여백/라운드는 `--bg*`, `--space-*`(8px 스케일), `--radius-*`(8/14/20/24) 토큰 사용. 하드코딩 hex 지양.
- 구 팔레트(네이비 액센트, Warm Sunrise 브라운/코랄, 그 이전 pink/teal)는 폐기.

## 모바일 UI 10대 원칙 (7번 하단내비 제외 → 실질 9개 적용)

1. **텍스트 가독성** — 심미성보다 가독성 우선. 1배수 최소 12pt+, 2배수 24pt+. 화면 작다고 폰트 줄이지 말 것.
2. **터치 가능한 버튼 사이즈** — 버튼 크기·간격 넉넉하게, 최소 44×44px. 터치 오류 방지.
3. **아이콘 통일감** — 스타일·톤앤매너 일관 유지. (이모지↔라인SVG 혼용 금지 등)
4. **명도 대비(접근성)** — 텍스트 너무 연하게 쓰지 말 것. 최소 4.5:1. (오렌지 CTA는 진한 글씨로 확보)
5. **디자인 기조 유지** — 직각/라운드 혼용 금지. 하나의 기조(라운드)로 통일.
6. **명확한 계층 구조** — 강-중-약 차등. 채움 CTA는 하나, 폰트는 컬러·굵기·크기로 강조 구분.
7. **효율적 하단 내비게이션** — 생소한 아이콘엔 텍스트 병기, 하단 아이콘 5개 이하. ※ **현재 웹 단계에서는 하단 탭 내비 자체를 제외**(앱 개발 단계로 이월).
8. **여유로운 여백** — 콘텐츠 간 여백 넓고 여유롭게. 스크롤 줄이려 여백 좁히지 말고 가독성 우선.
9. **정보 그룹핑** — 섹션·라인·섹션 간 여백·연한 배경면·카드 UI로 그룹핑.
10. **큼직한 레이아웃** — 좁은 모바일 화면 잘게 쪼개지 말 것. 큼직한 덩어리감으로 집중력 있게 전달.

## 적용 방법

- 대상: `index.html`, `login.html`, `mypage.html`, `admin.html`, 상세페이지(`challenge-*.html`).
- **375px 모바일에서 먼저 검증.** 로컬: `python -m http.server 5500`(python 미동작 시 정적 서버 대체).
- admin은 관리자 전용 데스크톱 도구라 밀도를 위해 터치(2번)는 실용적으로 적용(자주 누르는 컨트롤 위주), 가독성(1·4번)은 전면 적용.
