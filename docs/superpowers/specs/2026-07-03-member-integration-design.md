# 회원 시스템 통합 & 디자인 다듬기 — 설계

- 날짜: 2026-07-03
- 브랜치: `feature/full-redesign`
- 상태: 설계 확정, 구현 플랜 대기

## 목적

로그인/회원가입/마이페이지가 이미 Supabase로 붙어 있지만, 랜딩 사이트(index)와 **분리된 섬**으로 존재한다. 이걸 자연스럽게 통합해서 (1) 회원가입 동기를 자극하고 (2) 로그인/마이페이지 진입을 쉽게 만든다. 동시에 login/mypage/admin 페이지를 10대 모바일 UI 원칙에 맞춰 다듬는다.

## 최우선 제약: 모바일 퍼스트 (99% 모바일 유입)

> 유입의 99%가 모바일이다. 모든 결정은 모바일을 기준으로 한다.

- 모든 화면을 **375px 폭 기준으로 먼저** 디자인·검증한다. 데스크톱은 부차적.
- **터치 영역 최소 44×44px** (10대 원칙 2번).
- Phase 1 실사용 경험의 대부분은 **데스크톱 상단 내비가 아니라 햄버거 메뉴 + 모바일 어필 섹션**이다. 여기에 공을 들인다.
- admin도 내부 도구지만 모바일에서 동작·조작 가능한지 검증한다.

## 적용 원칙

- **10대 모바일 UI 원칙 중 7번(하단 탭 내비게이션)만 제외**, 나머지 9개 적용. 7번은 향후 앱 개발 단계로 이월.
- 색은 **Warm Sunrise 팔레트**(`tokens.css`): 브라운 텍스트(`--primary #241A12`), **코랄 CTA(`--action #C9471E`) = 신청/전환 전용**, 크림 배경(`--bg #FFFAF3`). (CLAUDE.md에 적힌 핑크/틸은 구버전 문서 — 별도로 갱신 필요.)
- 라운드 기조 유지(`--radius-*` 토큰), 명도 대비 4.5:1 이상.

## 아키텍처: 두 백엔드, 느슨한 연결

두 데이터 소스는 그대로 유지한다.

1. **신청** → Google Apps Script + 구글 시트(학생현황). **로그인 게이트 없음** — 누구나 신청 가능(전환 장벽 0).
2. **로그인/결과** → Supabase(members, daily_records, recordings). 마이페이지가 여기서 읽음.

**연결 방식:** 신청서는 전화번호가 키다. 회원(members)에 `phone`을 저장해두면 관리자가 "이 회원 = 이 신청자"를 전화번호로 매칭할 수 있다.

- 구글 로그인: 이메일 확보됨.
- 카카오 로그인: 닉네임만 확보(이메일 없음) → **가입 후 마이페이지에서 전화번호 1회 입력**받아 매칭 근거로 삼는다.
- `members` 테이블에 `phone` 컬럼 추가 필요(구현 플랜에서 확정).

이건 하드한 결합이 아니라 실무상 매칭이라 기술적으로 꼬이지 않는다.

## 구성 요소

### Phase 1 — index.html 통합 (최우선, 시각안 확정 완료)

**1-1. 세션 감지**
- index.html에 `<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2">` + `supabase-config.js` 추가.
- 페이지 로드 시 `MONC.getSession()`으로 로그인 여부 확인해 내비 상태를 전환.
- 로그인 상태면 `MONC.getMyProfile()`로 이름/아바타 이니셜 표시.

**1-2. 상단 내비 (B안)**
- **로그아웃:** `로그인/회원가입`(코랄 아웃라인 버튼) + `신청하기`(코랄 채움 CTA).
- **로그인:** 아바타(이름 첫 글자) + `마이페이지` 알약 + `신청하기`.
- 위계: 신청하기는 항상 유일한 채움 CTA. 로그인은 아웃라인이라 존재감은 있으나 CTA를 침범하지 않음(6번 원칙).
- **모바일 상단바에는 로그인 버튼을 넣지 않는다**(폭이 좁음) — 로고 + 햄버거만 유지.

**1-3. 모바일 햄버거 메뉴 (실사용 핵심)**
- **로그아웃:** 링크 4개(챌린지·강사진·커뮤니티·후기) + `로그인/회원가입`(코랄 아웃라인, 풀폭) + `챌린지 시작하기`(코랄 채움, 풀폭).
- **로그인:** 메뉴 최상단에 **회원 카드**(아바타 + "○○○ 님" + "마이페이지 가기 →", 크림/틴트 배경) → 링크들 → `챌린지 시작하기`.
- 모든 메뉴 항목 터치 영역 ≥44px.

**1-4. 회원 어필 섹션 (A안)**
- 위치: **후기 섹션 다음 · 마지막 신청 CTA 직전**(신청 직전 마지막 푸시).
- 헤드라인: "참여하면 **나만의 성장 기록**이 생겨요".
- 서브: 2주 매일 코치 피드백 + 수료 후 달라진 목소리 비교.
- **마이페이지 미리보기 목업**: 14일 진행 그리드(성공/아쉬움/미정 색) + Before/After 오디오 슬롯. 정적·예시용(실데이터 아님).
- 메인 CTA: `지금 신청하고 내 기록 시작하기`(코랄, 신청 모달 오픈).
- 보조 링크: "이미 참여하셨다면 로그인하면 바로 내 결과가 보여요"(login.html).
- **375px에서 먼저** 레이아웃 확정. 미리보기 목업이 좁은 화면에서 안 깨지게.

### Phase 2 — mypage.html 다듬기

이미 카드 구조는 양호. 10대 원칙 + 모바일 퍼스트로 폴리시.

- 인사말/헤더 위계 강화(강-중-약).
- 14일 그리드 셀 크기·터치 영역 확대, 색·아이콘 가독성 점검(4번).
- **Before/After를 감정적 하이라이트로 승격** — 변화가 회원의 핵심 보상.
- 빈 상태 문구 개선("아직 없어요" → 다음 행동 안내).
- 기수·입과일 등 메타 보강.
- **전화번호 입력 UI 추가**(아키텍처 연결용, 카카오 회원 매칭 근거).

### Phase 3 — login.html 다듬기

이미 깔끔. 최소 폴리시.

- 로그인 전 **"로그인하면 이런 걸 볼 수 있어요"** 짧은 안내(진행방식 · Before/After 언급)로 이탈 방지.
- 버튼 터치 영역·명도 대비 재확인, 라운드 기조 통일.

### Phase 4 — admin.html 다듬기 (우선순위 낮음)

내부 도구 → 가독성·효율 위주(마케팅 폴리시 불필요).

- 14일 편집 테이블 여백·정렬 정리.
- 저장 상태(dirty) 시각화 강화.
- 모바일 레이아웃·조작성 개선(사이드바 회원 리스트 ↔ 편집 전환 등).
- 계층 정리.

## 구현 순서

Phase 1(index 통합)이 가치가 가장 높고 시각안이 확정됐으니 먼저. 이후 2 → 3 → 4. 각 Phase는 독립 배포 가능.

## 검증

빌드/테스트 스위트가 없는 정적 사이트이므로 브라우저 렌더링으로 검증한다.

- **375px 모바일 뷰포트 우선** 검증(프리뷰 도구 `preview_resize mobile`).
- 로그아웃/로그인 두 상태 모두 내비·햄버거·어필 섹션 확인.
- 세션 감지 후 내비 전환이 깜빡임 없이 되는지.
- 터치 영역·명도 대비 점검.
- 데스크톱은 회귀 확인 수준.

## 범위 밖 (YAGNI)

- 하단 탭 내비게이션(앱 단계로 이월).
- 신청 로그인 게이트(신청은 계속 누구나).
- 신청 데이터를 Supabase로 이관(구글 시트 유지).
- 어필 섹션 목업에 실데이터 연동(정적 예시로 충분).
