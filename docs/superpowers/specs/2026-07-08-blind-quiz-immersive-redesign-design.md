# 면접관 체험(블라인드 퀴즈) 몰입형 리디자인 — 설계

날짜: 2026-07-08
대상: `index.html`의 `#blind-quiz` 섹션 (단독 수정, 다른 파일 영향 없음)

## 목표

현재 "흰 카드 + 아이콘 + 텍스트 + 버튼"의 평면적인 폼 느낌을,
**"내가 진짜 면접관이 되어 지원자를 심사하는"** 몰입형 장면으로 교체한다.
기존 5라운드 퀴즈 로직(랜덤 풀·점수·반전 카피 구조)은 그대로 재사용하고 UI 레이어만 바꾼다.

## 화면 구성 (3단계, 기존 intro/round/result 상태 구조 유지)

### ① 인트로 — "면접장 입장"
- 흰 카드 제거 → **사진 풀블리드 씬**: `bq-intro.jpg` + 하단 다크 스크림(그라데이션) + Ken Burns 슬로우 줌.
- 카피: 타이틀 "오늘, 당신이 면접관입니다" / 서브 "지원자의 두 목소리를 듣고 합격을 가르세요" / 메타 "🎧 5라운드 · 약 1분 · 이어폰 추천" / 오렌지 CTA "판정 시작하기".
- 사진이 가로형(16:9)이므로 모바일(375px)에서는 중앙 크롭(`object-fit: cover`) — 지원자 뒷모습 + 가운데 면접관들이 보이는 구도.

### ② 판정 라운드 — 실루엣 지원자 + 판정 콘솔
- 상단: **실루엣 영상 씬** (16:9 비율 유지, 라운드 모서리, muted/loop/playsinline, `preload="none"` — 퀴즈 시작 시 로드).
  - 영상 위 배지: 기본 "블라인드 심사 중" → 클립 재생 시 "지원자 A 답변 중" / "지원자 B 답변 중"으로 전환.
  - 상단에 진행 dots 5개 (현재 라운드 = 오렌지).
- 하단: 흰 **판정 콘솔 카드** — 지원자 A/B 플레이어 2개(재생 중 CSS 애니메이션 EQ 막대), 둘 다 1초 이상 들으면 판정 버튼 활성화 (기존 markListened 로직 재사용).
- **카피는 "지원자 A/B" 유지.** 실루엣은 익명(블라인드)이므로 한 명만 보여도 모순 없음 — 반전 시 "그래서 실루엣이 하나였다"로 시각적으로도 맞아떨어짐.
- 라벨 공개(reveal)·반전 카피·다음 라운드 흐름은 기존 로직 그대로.

### ③ 결과 — 기존 유지
- 점수·메시지·"나도 2주 뒤, 통과되는 목소리 만들기" CTA(`openApplicationModal()`)·다시하기 그대로.
- 결과 공유 카드(인스타 스토리 규격)는 **v2로 보류**.

## 모바일 디테일 (docs/design-principles.md 9대 원칙 준수)

- 375px 우선 검증. 터치 타깃 44px+, 사진/영상 위 텍스트는 반드시 스크림 위 흰 글씨(4.5:1 이상).
- 햅틱: 판정·정답 공개 순간 `navigator.vibrate()` 짧게 (미지원 브라우저 무시).
- `prefers-reduced-motion`: Ken Burns 정지 + 영상 자동재생 중단(정지 프레임 노출).
- 파형은 CSS 애니메이션 EQ (진짜 Web Audio AnalyserNode 파형은 iOS 이슈로 v2 보류).

## 기술 노트

- `index.html` 내 `.bq-*` CSS 블록과 `#blind-quiz` 마크업, 블라인드 퀴즈 IIFE의 UI 바인딩 부분만 수정.
- 오디오 풀(POOL_VOICE/POOL_SPIN)·셔플·점수·에러 폴백(`error` → markListened) 로직은 변경하지 않는다.
- `.bq-card [hidden] { display:none !important; }` 패턴 유지 (display:flex vs [hidden] 캐스케이드 이슈).
- 영상 로드 실패 시: 다크 네이비 배경 + 배지만으로 동작 (퀴즈 진행에 지장 없음).
