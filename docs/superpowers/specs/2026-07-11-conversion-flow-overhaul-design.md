# 홈페이지 구매전환 동선 개편 (모바일 우선, 전면)

- 작성일: 2026-07-11
- 대상: `index.html` (랜딩) 위주
- 배경: 모바일(375px) 실측 결과, 몰입 장치(체험·데모·후기)는 충분하나 그 몰입을 "신청 클릭"으로 잇는 동선이 곳곳에서 샌다. 재료를 새로 만들지 않고 **신청 동선과 오퍼를 제 위치에 배치**하는 것이 목표.

## 문제 정의 (모바일 실측 근거)

375×812 뷰포트 기준 실측:

- 총 페이지 높이 13,846px ≈ **17화면**.
- 진입 직후 **창문 인트로가 2,436px = 3.0화면** 순수 애니메이션. 이 구간에 스킵/신청 동선 없음.
- **상단 nav "신청하기" 버튼이 모바일에서 폭 0** (햄버거로 접힘) → 데스크톱의 상시 상단 CTA가 모바일엔 부재.
- **히어로 캐러셀(`.hero-scene`)에 신청 버튼 0개** (좌우 화살표뿐).
- 본문 직접 신청 앵커는 **docTop 92%·96% 두 곳뿐** — 중간 70%(Before/After·3단계·후기·챌린지 카드·연구진)에는 co-located 신청 동선 없음.
- 모바일 상시 신청 동선 = **하단 46px 고정바 하나**(폭 279px, 중앙 정렬).

## 스코프 결정

- 범위: **전면 개편** (모바일 4종 + 섹션 재배치 + 오퍼 상단 노출).
- 챌린지 카드: **상세페이지 유지 + '바로 신청' 보조 버튼 병기**.
- 시그니처 창문 인트로: **연출 길이 100% 보존 + '건너뛰기' 스킵 버튼만 추가**.

## 구조 전제 (구현 안전성)

- 스케일(zoom) 변환은 `scroll-fx.js`의 `initZoomExit`가 `[data-zoom-exit]` 래퍼 내부의 `.zoom-exit-pin`에만 적용. **`.hero-window-intro`만 해당**.
- `.hero-scene`(히어로 캐러셀)의 `.zoom-exit-pin`은 `data-zoom-exit` 래퍼가 아니므로 **JS 스케일을 받지 않는 일반 sticky** → 내부에 CTA를 안전하게 얹을 수 있음.
- 인트로 스킵 버튼은 태그라인(`.hero-tagline`)과 동일하게 **스케일 핀 밖(형제)**에 두어 확대를 받지 않게 한다.
- 가격/보증금/계좌/제출 스키마는 **불변** → 휴면 모달 2곳·`apply.html` 이중관리 대상 아님. 카드 '바로 신청'은 기존 `apply.html?c=<id>` 프리셀렉트를 재사용.

## 변경 항목

### A. 인트로 스킵 버튼
- 위치: `.hero-window-intro` 내부, 스케일 핀 밖. 우하단 pill `건너뛰기 ↓`.
- 동작: 클릭 → `#home`(히어로 캐러셀)으로 부드럽게 스크롤. 무JS/`prefers-reduced-motion`에서도 `href="#home"` 폴백으로 동작.
- 노출: 인트로 진행 중에만 표시하고, **줌 진행률 ≥ 0.85(= 인트로 거의 통과) 시점에 숨김**(`monc:zoomprogress` 이벤트 구독, 미수신 시 스크롤 위치 폴백). 모바일 하단 고정바(46px)에 가리지 않게 `bottom` 보정.

### B. 히어로 캐러셀 신청 CTA
- 위치: `.hs-carousel` 내부, 슬라이드 4개 공통 CTA바 1개(`.hs-nav` 뒤).
- 내용: 주 버튼 `신청하기 →` → `apply.html`, 보조문구 `참가비 3만원 · 2주 완주 시 전액 환급`.
- 노출: 데스크톱·모바일 공통. 슬라이드3(크림 배경) 대비색만 오버라이드.

### C. 하단 고정 CTA바 강화
- 현재 279px 중앙 정렬 → **full-width**, 높이 ≥52px(터치 여유).
- 문구 2줄 유지(`지금 신청하기` / `참가비 3만원 · 수료 시 전액 환급`).
- `recruit.js` 모집상태 연동 시 긴급성 뱃지(`모집 중 · D-N`, 마감 시 톤 전환)를 서브텍스트에 반영. 데이터 없으면 기존 문구 폴백.
- `aria-label` 갱신.

### D. 감정 피크 CTA 병기
- Before/After(`#before-after`) 섹션 끝: 버튼 `나도 이렇게 바뀌기 →` → `apply.html`.
- 커뮤니티(`#community`) 섹션: 기존 `후기 더 보기 →`(reviews.html) 옆/아래에 `지금 신청하기` 병기(이탈만 있던 구간에 전환 동선 추가).

### E. 섹션 재배치
- `advisor`(진단, `#advisor`)를 `challenges`(`#challenges`) **바로 위로** 이동.
- 결과 순서: … `community` → `advisor` → `challenges` → `instructors` → `member-appeal` → `cta`.
- `advisor` IIFE·`recruit.js`는 셀렉터 기반이라 순서 무관(마크업 블록만 이동).

### F. 카드 '바로 신청' 병기
- 각 `.challenge-card`에 보조 버튼 `바로 신청` → `apply.html?c=<recruit-id>`.
- 앵커 중첩 방지: 카드 `<a>` → `<div class="challenge-card">`로 변경 후 **stretched-link 패턴** 적용(상세페이지 = 카드 전체 클릭, 바로신청 = 별도 `<a>` 버튼이 stretched-link 위에 위치).
- `recruit.js`는 `.challenge-card` 클래스로 대상을 잡으므로 태그 변경 무관. 모집 마감 시 '바로 신청' 버튼 비활성 처리(`.recruit-status`/닫힘 로직과 일관).

### G. 오퍼 상단 노출
- B(히어로 CTA 보조문구) + C(하단바)가 `3만원·전액환급`을 상단·상시로 노출 → 별도 리본 없이 충족(클러터 방지, YAGNI).

## 비목표 (YAGNI)

- 가격/보증금/계좌/커리큘럼/제출 스키마 변경 없음.
- 인트로 연출 자체(줌 커브·타이밍·조립) 변경 없음 — 스킵 버튼만 얹음.
- 휴면 모달(`application-modal.js`·index 인라인) 정리는 이번 범위 밖.
- 후기/커뮤니티 데이터 소스 변경 없음.

## 검증 방법

- 프리뷰 스크린샷은 무거운 히어로 때문에 얼어붙음 → **DOM 실측(`getBoundingClientRect`)·computed-style 재측정**으로 CTA 존재/위치/노출 확인.
- `read_console_messages`·`preview_logs`로 JS 에러 없음 확인.
- 모바일(375px)·데스크톱 각각 레이아웃 측정: 히어로 CTA inViewport, 하단바 full-width, 카드 '바로 신청' 앵커 존재, 섹션 재배치 순서(docTop) 확인.
- 배포 후 오너 실기기 육안 확인(모바일 유입 99%).

## 리스크

- 카드 `<a>`→`<div>` 전환 시 기존 CSS 셀렉터(`a.challenge-card` 등 태그 결합 셀렉터)가 있으면 스타일 깨질 수 있음 → 구현 시 태그 결합 셀렉터 존재 여부 확인 후 클래스 기반으로 정리.
- 히어로 캐러셀 CTA가 sticky 핀 레이아웃/높이(812px)에 영향 주지 않도록 배치 검증.
- 인트로 스킵 버튼이 태그라인 fixed 승격 로직과 z-index 충돌 없게 배치.
