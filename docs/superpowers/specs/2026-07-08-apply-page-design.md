# 신청·결제 전용 페이지 (`apply.html`) 설계

작성일: 2026-07-08

## 배경 / 목표
기존 신청·결제는 **팝업 모달**(index.html 인라인 + application-modal.js, 두 곳 중복)로 처리됐다.
이를 SaaS 요금제 페이지 같은 **상세 페이지 형식**으로 바꿔 구매 전환을 높인다.
- 각 챌린지 커리큘럼·설명은 그대로 유지(카드 선택 시 아코디언으로 펼침)
- FAQ로 구매 전 불안 해소
- 회원가입 유도 배너로 회원 전환도 함께 유도

## 범위
- 신규 페이지 `apply.html` 하나 생성 (모바일 우선, `tokens.css` 재사용)
- index.html + 4개 상세페이지의 "신청하기" CTA → 전부 `apply.html`로 이동
- 결제 방식은 **현행 그대로 계좌이체 + Supabase `applications` 저장** (토스는 오너가 나중에)
- 기존 팝업 코드(application-modal.js, index 인라인 모달)는 **삭제하지 않고 유지**, CTA 링크만 교체

## 페이지 구조 (위 → 아래)
1. **히어로** — "2주 뒤, 통과되는 나" + 신뢰 배지(수료 시 보증금 환급 / 실제 합격 후기)
2. **챌린지 카드 4개** (보신각·영합각·스피닝·승자각)
   - 카드 클릭 = ⓐ 선택(장바구니식, 다중 선택) + ⓑ 그 아래 2주 커리큘럼 아코디언 펼침
   - 다시 클릭하면 선택 해제 + 접힘
   - 각 카드: 참가비 3만 + 보증금 3만 = 6만원 (할인 없음)
   - 커리큘럼 내용은 기존 application-modal.js 것 그대로 이식
   - `recruit.js`의 `loadChallengeStatuses()`로 마감/모집예정 챌린지 카드 자동 비활성
3. **회원가입 유도 배너** (챌린지 카드 바로 아래, 네이비 톤으로 구분)
   - "잠깐! 회원가입하면 2주간 성장한 내 모습을 매일 확인!" → `login.html` 연결
4. **조합 추천 배너** — "목소리+답변 함께" 넛지 (할인 없음, 정보성)
5. **FAQ 아코디언** (아래 문항 참조)
6. **신청·결제 폼** — 이름 · 전화 · 환급계좌 + 선택 요약 + 금액(참가비/보증금/합계) + 신한 계좌 안내(복사 버튼) + 신청하기
   - 제출: `window.MONC.sb.from('applications').insert({name, phone, refund_account, challenges[], total_price})` — application-modal.js와 동일 스키마
   - 라운드 정보는 `loadChallengeStatuses()` 후 `window._challengeRounds` 사용
   - 중복 제출 방지 플래그(`_isSubmitting`) 유지
7. **하단 고정 바** — "선택 N개 · N만원 · 신청하기" (스크롤 내내 고정, 클릭 시 6번 폼으로 스크롤/제출)

| 5 | 결제는 어떻게 하나요? | 신한 계좌 입금 후 신청 버튼을 눌러주세요 |
