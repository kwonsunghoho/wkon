# 랜딩 텍스트 다이어트 — 재편 시안 (구조 정리 1차 + 안전 2차)

날짜: 2026-07-14
대상: `index.html`(홈)만. apply.html 신청폼·가격·법적 필수 동의는 손대지 않음.

## 배경 / 진단
오너 진단: "텍스트가 너무 많아 산만·지저분함. 톤·분위기는 유지하며 최소화하고 싶다."
근본 원인 3가지(문장 길이가 아니라):
1. **같은 메시지 반복** — '같은 사람의 2주 변화'가 블라인드퀴즈→B&A→커뮤니티 3섹션 연속, 히어로 카드 4장 서브라인 동일, 강사 수치 3중복, 환급 문구 3연속.
2. **헤더 3~4겹 적층** — eyebrow+제목+리드문이 겹치며 제목·리드문이 같은 말.
3. **신청 CTA 동사 5종 난립.**

## 오너 결정
- **범위 = 1차(구조 정리) + 2차 '추가 안전' 8곳.** (2차 '추가 과감'·'취향/모험' 티어는 보류.)
- **강사진(#instructors) 이력은 전부 유지(성역).** 1차에서 검토했던 강사 bio 첫 문장 삭제·세부 이력 토글 이관은 **원복**. 개인 경력·수치·핵심이력·괄호 세부 불가.
- **상한선(넘지 않음):** 히어로 팩트라인·모바일바 서브라인 중 최소 1개(환급 상주), 커뮤니티 브릿지('특별한 사례가 아니에요'), 퀴즈 저분기 위로 문구('합격은 그 미묘한 지점에서 갈립니다'), 상품 팩트(환급·기간)를 0회로 만들지 않기.

## 반영 항목

### 1차 · 구조 정리
- **히어로(#home):** 카드 4장 공통 `.hs-sub` "MONC 챌린지가 전과 후로 증명해요" 삭제 / `.hs-cta-sub`에서 '참가비' 제거.
- **연구진 스트립(#researchers-strip):** 제목 2줄→1줄 명사구 "전·현직 승무원 & 면접관 출신 연구진"(상투구 삭제) / 리드문 "분야마다 전담 연구원이 설계합니다."(4항목 나열 삭제) / CTA "연구진 프로필 보기 →"('자세히' 제거).
- **블라인드 퀴즈(#blind-quiz):** 헤더 리드문 1줄로("면접관의 귀로 직접 판정해 보세요.") / 인트로 서브라인 첫 절 삭제("당신이라면, 누구를 통과시키시겠어요?") / 라운드 반전 문구 매 라운드→1라운드 1회(idx===0 분기, 2~5라운드 확인 문구) / 결과 punch 삭제(4겹→3겹) / 결과 메시지 최저·최고 군더더기 제거.
- **Before&After(#before-after):** 리드문 1문장으로("모두 같은 사람의 단 2주 전후입니다.") / 영합각 카드 인용 "2주 전후가 영상 그대로 담겼어요.".
- **커뮤니티(#community):** 라이트박스 힌트 1줄로("누르면 확대 · 바깥을 누르면 닫힘").
- **진단(#advisor):** 리드문 "세 가지 질문이면, 나에게 맞는 챌린지를 알려드려요."('지금 나에게 딱 맞는 챌린지' 결과 카드에만).
- **챌린지 목록(#challenges):** 리드문 "이름을 눌러 미리 만나보세요." / 혜택형 유사버튼 4개(.challenge-action) 삭제 → 카드당 '바로 신청 →' 단일 동선.
- **성장기록(#member-appeal):** 리드문 "매일 코치 피드백을 받고, 달라진 내 목소리를 직접 비교해요." / 재방문 안내 "이미 참여하셨다면 로그인해 바로 확인하세요.".
- **최종 CTA:** desc 첫 문장 삭제 / note "· 부담 없이 시작해요" 삭제 / 마퀴 '2주 완주 시 보증금 전액 환급' 항목 삭제(두 span 동일).
- **신청 CTA 라벨 통일 → '신청하기':** 모바일메뉴·성장기록·최종CTA·모바일하단바 메인. (nav·히어로 이미 '신청하기', 조합추천·카드 '바로 신청'은 제외.)

### 2차 · 추가 안전 8곳
1. 연구진 스트립 `.section-label` 'Instructors' 삭제(강사진 섹션과 중복).
2. 커뮤니티 `.mc-stat-label` '챌린지 참여자/수료율' → '참여자/수료율'.
3. 퀴즈 결과 `.bq-score` '5라운드 판정 완료' 삭제.
4. 퀴즈 `.bq-retry` '다른 목소리로 다시 판정하기' → '다시 판정하기'.
5. 모바일 메뉴 `.mm-quiz-tag` '1분 블라인드 테스트' 삭제.
6. 성장기록 `.wf-label` 'Before · 입과 전 / After · 수료 후' → 'Before / After'.
7. 성장기록 `.wf-hint` '또는 파일을 여기로 드래그' 삭제(두 패널).
8. 퀴즈 인트로 `.bq-intro-badge` '면접장 입장' 삭제.

## 구현 주의(QA)
- 히어로 `.hs-sub` 4장 삭제 후 카드 세로 여백·coverflow 높이 4장 동일 확인(heroLayout 인라인 스타일).
- 퀴즈 explain 1라운드 분기·결과 punch 삭제 시 JS 매핑·리셋 idx=0 복귀 확인. 저분기 위로 문구는 유지.
- 최종 CTA 마퀴 두 span 동일 수정(무한 흐름 복제 — 한쪽만 고치면 간격 틀어짐).
- 커뮤니티 라이트박스 힌트 통합 시 상태 전환 JS가 사라진 span 참조 안 하는지.
- CTA 라벨 수정 시 동일 문자열이 다른 곳(조합추천 3152·휴면 모달 4689)에 있는지 grep 확인 — 그 둘은 건드리지 않음.
- 커뮤니티 `.mc-stat-label`·퀴즈 결과·진단 등이 JS 렌더/폴백인지 확인해 소스(마크업 or JS 상수) 정확히 수정.
- 검증: 375px 모바일 우선(히어로 프리뷰 무거움 — DOM/계산 스타일 기반 검증).

## 리파인 5종 (같은 날 오후, 오너 피드백)
텍스트 다이어트 위에 얹은 구조 개선. 방향은 시안(전/후 + 목업)으로 확정.

- **#1 블라인드 퀴즈 인트로 축소**: 오버레이 배지·서브라인 제거, 제목 + '판정 시작하기'만. 메타 '🎧 약 1분 · 이어폰 권장'로 축약.
- **#2 커뮤니티 후기 카드 = A안**(오너 선택): 앞면 스크린샷 캡처 밴드 폐기 → '실제 후기' 배지 + 명조 한줄평(`.mc-review-quote` serif) + 이름·챌린지 칩 + '원문 보기'(`.mc-review-src`) 라이트박스. 지저분한 잔글씨는 라이트박스로만.
- **#3 챌린지 진단 접기**: 평소 `.adv-intro`(시작 버튼)만, 클릭 시 `.adv.is-started`로 문항 펼침(`paintStep(0)`). 구 '즉시 Q1' 폐기.
- **#4 연구원 상세 → `researchers.html` 분리**(오너 선택 = 별도 페이지): 랜딩 `#instructors` 삭제, 전용 페이지 신설(연구원 5명·탭·이력 그대로). 링크 3곳(스트립 CTA·nav·모바일) → researchers.html. index 잔존 `researchers`/`.mi-*`/IIFE는 가드로 no-op.
- **#5 성장기록 대시보드 = 성장 리포트형**(오너 선택): 구 `.ma-preview`(도장+파형) → 진행 링·스탯·성취 배지 중심 `.db-shell` 리포트. B&A 파형은 '변화 비교' 보조로 강등, 입체 레이어. 구 maDays/waveform IIFE는 `#maPreview` 제거로 무력화.

검증: 모든 페이지 콘솔 오류 0·가로 넘침 0·섹션 순서 정상·강사진 이력 원본 유지·라이트박스/진단 시작/링크 동작 확인.
