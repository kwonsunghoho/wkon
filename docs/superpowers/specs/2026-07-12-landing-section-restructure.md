# 랜딩(index.html) 섹션 개편 계획 — 2026-07-12

> 홈에서 논의·확정한 개편안. 회사에서 이 문서를 열고 체크리스트 순서대로 이어서 작업하면 됨.
> **아직 코드는 하나도 안 바뀐 상태** — 이 문서가 유일한 산출물.

## 배경: 현재 섹션 구성 진단 (82/100)

현재 순서:
창문 인트로 → 히어로 캐러셀(#home) → 블라인드 퀴즈(#blind-quiz) → Before&After(#before-after) → **3단계 How(#how)** → 커뮤니티(#community) → 챌린지 진단(#advisor) → 챌린지 목록(#challenges) → 강사진(#instructors) → 성장기록(#member-appeal) → 최종 CTA

잘된 것: 퀴즈→B&A(문제 체감→증거) 페어, Advisor→챌린지 인접, 증거마다 CTA 반복, 환급 리마인드 마무리.

감점 4가지:
1. **How 위치** — STEP 01이 "챌린지 선택"인데 챌린지 소개는 세 섹션 뒤. 동시에 두 증거 섹션(B&A·커뮤니티)을 갈라놓음. → **오너 결정: 삭제**
2. **증거 분산** — B&A와 커뮤니티가 How를 사이에 두고 분리, 누적 효과 끊김.
3. **상품 인지 늦음** — 챌린지 목록이 11개 중 8번째. 특히 **보증금 전액 환급 USP가 페이지 맨 끝(최종 CTA 노트·마퀴)에서야 첫 등장**.
4. **이중 히어로 스크롤 비용** — 인트로(러웨이 390vh)+히어로 연속, 모바일 첫 실콘텐츠 도달까지 김. → **오너 결정: 인트로 연출은 유지, 비용만 절감**

목표 순서(예상 87~88점):
인트로 → 히어로 → 퀴즈 → B&A → 커뮤니티 → Advisor → 챌린지 → 강사진 → 성장기록 → CTA

## 작업 체크리스트

### 1. How 섹션 삭제 (확정)
- [ ] `index.html` `<!-- HOW -->` 주석 포함 `<section class="how" id="how">` 전체 삭제 (작성 시점 기준 2259~2294행)
- [ ] 푸터 "서비스 소개" `href="#how"`(2584행) → `#challenges`로 변경
- [ ] CSS 정리: `.how` `.how-grid` `.step-card` `.step-icon` `.step-num` `.step-title` `.step-desc` — 데스크톱(377행 부근) + 모바일 미디어쿼리(1471~1473, 1509행 부근)
- [ ] 스프라이트 심볼 `#i-target`(1903행)·`#i-mic`·`#i-sparkles`(1909행 부근)는 **How에서만 사용 확인됨**(index 내). 다른 html에서 안 쓰면 삭제 가능, 놔둬도 무해 — 저위험 원하면 유지.
- 메시지 손실 없음: "준비물·복잡한 과정 없음"은 최종 CTA("준비 없이도 괜찮아요")와 apply.html FAQ #3(진행방식)이 커버.
- ⚠️ 라인 번호는 삭제 진행하며 밀리니 grep으로 재확인하며 작업.

### 2. 증거 연타 마감 (B&A → 커뮤니티가 인접해진 뒤)
- [ ] **브릿지 카피**: 커뮤니티 `mc-title`("혼자 하지 않아도 돼요", 2308행 부근) 아래에 연결 문장 추가 — 예: "방금 들으신 변화, 특별한 사례가 아니에요." 직후 참여자 수 카운트업이 이어져 깊은 증거→넓은 증거로 쌓이는 구조.
- [ ] **연속 CTA 정리**: B&A 끝 "나도 이렇게 바뀌기 →"(apply, 2254행)는 유지(전/후 음성 직후 = 감정 피크). 커뮤니티 하단(2329행) `btn-action` "지금 신청하기"는 제거 또는 고스트 강등하고 "후기 더 보기 →"를 주 동선으로.

### 3. 상품 선공개 (섹션 이동 없이 카피 레벨)
- [ ] **히어로 팩트 라인**: `.hs-cta` 근처에 "2주 · 하루 한 번 미션 · 완주 시 보증금 전액 환급" 한 줄 추가. 환급 USP 첫 화면 선공개가 핵심. (`.hs-cta`엔 이미 `data-recruit-cta-badge` D-day 뱃지 있음 — 시각 위계 겹치지 않게)
- [ ] **중간 CTA 목적어**: "지금 신청하기"류 → "2주 챌린지 신청하기" 등 뭘 신청하는지 문구에 담기.
- 참고: apply.html이 카드+커리큘럼+가격을 재소개하는 완결형 페이지라 조기 신청 클릭이 이탈로 안 이어짐 → 구조 재배치 불필요.

### 4. 창문 인트로 비용 절감 (연출 유지 — 러웨이 축소 금지)
- [ ] **재방문 단축**: `localStorage`(예: `monc_intro_seen`)로 최초 시청 기록 → 재방문 시 러웨이 대폭 축소 또는 조립 완료 MONC 정적 한 컷 후 바로 히어로. ⚠️ scroll-fx.js가 init에서 `data-zoom-*` attr을 캐시하므로 **init 전에** attr을 바꿔야 함. `calibrateZoom()`/`scanOpening()` 실측 로직과 충돌 없는지 확인.
- [ ] **건너뛰기 어포던스**: 인트로 구석에 작은 텍스트 버튼("건너뛰기") → `#home` 스무스 스크롤. 첫 방문 연출은 그대로, 급한 사람만 탈출.
- [ ] **계측 후 판단**: 첫 방문자의 히어로(#home) 도달률 측정 — Supabase 테이블(예: `page_events`) insert 비콘 정도로 가볍게. 수치가 나쁠 때만 러웨이 조정 논의.
- 이미 우회되는 경로(손댈 것 없음): reduced-motion 정적 폴백, 앵커 딥링크 유입.
