# 2026-07-12 — 인트로 MONC 로고 전진(줌 동반 확대) 설계

## 배경 / 목표

- 현재 창문 인트로: 조립(0~30%) 후 MONC 락업은 **크기 고정**으로 잔류, 창문 줌만 확대 → 통과(~89%) → 여운(89~94%) → 페이드(94~99.5%).
- 오너 요청: 하늘로 빨려들어가는 줌 구간에서 **MONC 글자도 점점 커지며 앞으로 다가오는 느낌**을 추가.
- 구 '확대+페이드 동반 퇴장(52~78%)'의 폐기 사유는 **조기 페이드**(통과 전에 로고가 사라짐 → 브랜드 여운 소멸)였지 확대 자체가 아님. 이번엔 **페이드 타이밍(94~99.5%)을 건드리지 않고 확대만 재도입**한다.

## 설계

1. **scroll-fx.js `renderWrap`**: 매 프레임 계산하는 핀 배율(pinScale)을 `monc:zoomprogress` 이벤트 detail에 `scale`로 동봉. 줌 커브·개구부 캘리브레이션·6.5캡의 단일 소스를 renderWrap 하나로 유지하고, 소비자는 프레임 단위로 정확히 동기화된다.
2. **태그라인 IIFE (index.html)**: `GROW_ALPHA = 0.30`. 매 프레임 `g = pinScale^0.30`을 `.ht-inner`(MONC 로고+서브라인 락업)에 `transform: scale(g)`로 적용. 핀 최종 배율 4~6.5 기준 최종 **~1.5–1.8×**.
   - `.ht-inner`는 구 연출의 `will-change: transform`이 살아 있어 합성 승격 상태 — transform 한 줄 갱신, 리플로우 없음(성능 계약 준수).
   - `lastG` 가드로 동일값 재기록 스킵. `measure()`가 inner transform을 리셋하므로 그때 `lastG = -1`로 재무장.
   - 이벤트 미수신(scroll-fx 로드 실패 폴백)·reduced-motion·구버전 scroll-fx 캐시 조합 = `lastScale`이 1로 남아 **전진 없이 기존 동작** (우아한 강등).
3. **러웨이 390/340 유지** — "계측 데이터 없이 축소 금지" 가드 준수(오너 확정). 전진 연출로 체감 속도가 얼마나 빨라지는지 본 뒤 page_events 데이터로 축소를 재검토한다.

## 왜 0.3승(로그 공간 비율)인가

줌의 체감 속도는 배율의 '비율' 변화라서, 로고 배율을 핀 배율의 거듭제곱(`pinScale^α`)으로 잡으면 창틀(가까움, 빠름)과 로고(멂, 느림)가 **같은 카메라 이동의 원근 시차**로 읽힌다. 조립 직후(zp=0)엔 정확히 1.0이라 네이티브 렌더 교체(선명도)의 전제도 유지된다.

## 안전성 검토

- **창틀 침범 없음**: 개구부는 `(1+K)^(zp^1.35)`, 로고는 그 0.3승 — 개구부가 전 구간 항상 더 빨리 커지므로 조립 시 fit(개구부 폭 88%)이 끝까지 유지된다.
- **모바일(375px)**: 로고 폭 ~200px → 여운 시점(~89%) 약 1.45× ≈ 290px, 페이드 종점 ~1.6× ≈ 320px — 화면 안. 여운 구간의 브랜드 각인이 잘리지 않는다.
- **선명도**: 확대 시작점이 정지 상태(scale 1, 네이티브 렌더)라 정적 구간은 픽셀 퍼펙트. 확대 중엔 이동+모바일 DPR≥2로 래스터 여유가 있어 흐림 체감 없음(DPR1 데스크톱 끝자락은 페이드 중이라 수용).
- **불변**: 조립 수학(measure는 transform 리셋 후 측정), 스크림(50~75%)·창틀(82~90%)·하늘(96~100%)·락업(94~99.5%) 페이드, 여운 구간, 건너뛰기/계측 IIFE(detail.progress만 사용).

- 러웨이: **유지 (390/340, 재방문 200/180)**

## v2 — 자체 커브로 교체 (2026-07-12 당일, 오너 피드백)

v1 로컬 확인에서 오너 피드백: **"MONC 크기 그대로인데? 앞으로 다가오는 느낌이 없는데?"**

### 원인 (수학)

v1 `g = 핀배율^0.3`은 핀 커브 `(1+K)^(zp^1.35)`의 **후반 몰림을 그대로 물려받는다**(로그 공간 0.3배는 커브 모양 불변). 1+K=5 기준:

| 진행률 p | v1 (핀^0.3) | v2 (1.7^(zp^0.7)) |
|---|---|---|
| 0.40 | +3.6% | +14.6% |
| 0.50 | +9.3% | +24.7% |
| 0.60 | +16.6% | +34.1% |
| 0.89 (통과) | ×1.47 | ×1.60 |
| 1.00 | ×1.62 | ×1.70 |

인트로 절반까지 +9%는 옆에서 창틀이 +35% 커지는 동안이라 지각 불가. **체감을 결정하는 건 종점 배율이 아니라 여정 중간의 변화율**이다.

### v2 설계

- `g = GROW_END^(zp^GROW_SHAPE)` = `1.7^(zp^0.7)` — 지수 β=0.7(<1)로 초중반이 두터워 조립 직후부터 다가옴이 체감되고, 후반 몰림인 창틀 줌과 리듬이 엇갈려 "로고는 꾸준히, 창틀은 막판 가속" 대비가 생긴다.
- **핀 배율과 분리, 종점 K 무관 고정** — 모바일 여운(89~94%) 로고 ~330px<375px fit 보장, 울트라와이드 과확대 없음. scroll-fx의 detail.scale 동봉은 소비자가 없어져 **회수**(이벤트는 progress만).
- 부수 효과로 잠재 버그 차단: v1은 measure() 도중 calibrateZoom→`monc:zoomcalib`→renderWrap→`monc:zoomprogress` **동기 재진입** 시 새 K로 달라진 scale이 lastG 가드를 뚫고 inner에 transform을 써 FLIP 측정 좌표를 오염시킬 수 있었다. v2는 g가 p만의 함수라 재진입(같은 p)에서 가드 스킵이 보장된다.
- 안전성 재검증: 개구부 성장 대비 로고 일시 초과 최대 +5.6%(K 폴백 3.2 최악) < fit 여유 13.6% — 창틀 침범 없음.

### 튜닝 (2026-07-12 당일, 오너: "확 다가오는게 느껴지게")

`GROW_SHAPE` 0.7 → **0.55**. 조립 직후 상승률이 크게 올라간다: p=0.35에서 +8.7%→+13.2%, p=0.5에서 +24.7%→+30.5%, 통과(0.89) ×1.60→×1.62. 종점은 ×1.7 그대로(모바일 여운 ~316px<375px). **β 0.5 미만 금지** — 개구부 성장 대비 일시 초과가 β=0.5(K 폴백 최악)에서 +12.5%로 fit 여유 13.6%를 거의 소진한다(0.55는 +10.4%로 여유 유지, 0.88 fit에는 텍스트 그림자 블리드 예산도 포함되어 있음).

### 교훈

동반 연출의 배율을 주 줌의 거듭제곱으로 잡으면 주 줌이 back-loaded일 때 조연도 back-loaded가 된다 — **조연의 체감 커브는 자체 설계할 것.**
